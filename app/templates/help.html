AWS Permissions Help

To use the AWS Cost Explorer Dashboard, you need an AWS IAM user with specific permissions to read cost and usage data. We recommend creating a Read-Only user for maximum security. This dashboard displays daily costs, service breakdowns, region breakdowns, and detailed resource usage.

1. Create an IAM User

Go to the AWS Console, navigate to IAM, and click Create User. Provide a name and select Provide user access to the AWS Management Console (optional).

2. Attach Permissions Policy

Choose Attach policies directly and then Create policy. Use the JSON editor to paste the following policy which allows read-only access to AWS Cost Explorer data: 

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "ce:GetCostAndUsage",
                "ce:GetDimensionValues",
                "ce:GetCostForecast",
                "ce:GetUsageForecast"
            ],
            "Resource": "*"
        }
    ]
}

What these permissions allow:

  • ce:GetCostAndUsage - Retrieve cost and usage data (required for all dashboard features)
  • ce:GetDimensionValues - Get available regions and services
  • ce:GetCostForecast - Access cost forecasting data (optional)
  • ce:GetUsageForecast - Access usage forecasting data (optional)

3. Generate Access Keys

Once the user is created, go to the Security credentials tab and click Create access key. Choose Third-party service or Other as the use case.

4. Use your keys

Copy your Access Key ID and Secret Access Key and enter them in the login modal to view your AWS Cost Explorer dashboard with:

  • Daily cost trends (last 7 days)
  • Service-wise cost breakdown
  • Region-wise cost breakdown
  • Detailed region-service breakdown table
  • Resource usage tracking
